# Thread E: 포털 자동 송고(XML/RSS) 모듈

## 목표
- 네이버/다음 등 포털 요구 규격에 맞는 피드를 제공하고, 갱신/정정/회수 이벤트를 안정적으로 반영한다.

## 범위
- 포털별 XML/RSS 어댑터(`naver`, `daum`, `google-news-feed` 분리)
- `lastBuildDate` 갱신 로직
- ping/재전송/실패 추적
- 송고 성공률/지연 관측
- 변경분 checksum 기반 재송고 제어

## 제외 범위
- 포털 제휴/정책 계약 업무
- 독자용 웹 UI

## 데이터 계약
- 입력 이벤트: `article.published`, `article.updated`, `article.retracted`
- 출력 산출물:
  - 포털별 피드 URL
  - 포털별 어댑터 렌더 결과(XML payload)
  - 송고 로그(`portal_delivery_logs`)
  - 실패 재처리 큐
  - 기사별 최종 송고 버전/체크섬(`article_portal_states`)

## 어댑터 정책 (고정)
- 변경 없는 반복 송고 금지: `content_checksum` 동일 시 송고 생략
- 실패 무한 재송고 금지: 최대 재시도/백오프/쿨다운 이후 수동 전환
- 수정/삭제 즉시 반영: `updated/retracted` 이벤트는 우선순위 큐로 처리
- 동일 URL 전면 덮어쓰기 금지: 대폭 변경은 L3 정책(신규 기사 + 관계 연결) 준수

## 작업 항목
1. 포털별 피드 필드 매핑 정의
2. 포털별 어댑터 인터페이스/구현(`render`, `validate`, `send`, `parse_response`)
3. XML 생성기/검증기 구현
4. 테스트 송고 모드/샌드박스 연동
5. ping 엔드포인트 호출기 구현
6. 실패 재시도/백오프/수동 재송고 기능
7. 송고 결과 대시보드 및 알림 연동

## 완료 기준 (DoD)
- 표준 샘플 검증 통과(필수 필드/인코딩/날짜 형식)
- 발행/정정/회수 이벤트 반영 지연이 목표 이내
- 실패 건 재처리 성공률 기준 충족
- 포털별 에러 유형 분류 가능
- 변경 없는 반복 송고가 시스템 레벨에서 차단됨

## 검토 체크리스트
- UTF-8/CDATA/특수문자 이스케이프가 안전한가
- 동시 수정 시 마지막 버전만 노출되는가
- 회수 기사 처리(삭제/대체 안내)가 정책과 일치하는가
- ping 실패 시 누락 없이 재시도되는가
- 실패 상태 누적 시 자동 송고 정지(서킷 브레이커)가 동작하는가
