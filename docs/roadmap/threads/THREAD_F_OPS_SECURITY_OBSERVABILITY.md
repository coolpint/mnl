# Thread F: 운영/보안/관측성

## 목표
- 뉴스룸 서비스 운영 안정성과 보안, 비용 가시성을 확보한다.

## 범위
- 인증/권한 보안 강화(2FA 권장, 최소 권한 원칙)
- 비밀키 관리 및 로테이션 정책
- 로그/메트릭/트레이스 수집
- 백업/복구/롤백 런북
- 비용 대시보드(LLM/API/인프라)

## 제외 범위
- 기능 개발 자체
- 디자인/UI 개선

## 핵심 SLI/SLO (초안)
- 발행 지연 시간 p95
- AI 초안 생성 성공률
- 포털 송고 성공률
- CMS 에러율
- 월간 인프라/LLM 비용 상한

## 작업 항목
1. 중앙 로그 스키마 정의(요청 ID, 기사 ID, 잡 ID)
2. 알림 룰 설정(에러율, 재시도 누적, 송고 실패)
3. 비밀키/환경변수 관리 체계 수립
4. DB 백업/복구 리허설
5. 장애 대응 런북 작성
6. 비용 추적 태깅 및 월간 리포트 자동화

## 완료 기준 (DoD)
- 주요 실패 시나리오에서 탐지/알림이 동작
- 복구 절차가 문서 + 리허설로 검증됨
- 비밀키가 코드/로그에 노출되지 않음
- 비용 경보 임계치가 설정됨

## 검토 체크리스트
- 관리자 계정 보호(2FA, IP 제한)가 적용되는가
- 감사 로그 보존 기간/조회 권한이 정의되었는가
- 장애 시 수동 발행 우회 절차가 있는가
- 모델/API 장애 시 대체 경로가 있는가
