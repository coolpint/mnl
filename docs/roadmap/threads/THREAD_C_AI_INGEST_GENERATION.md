# Thread C: AI 인제스트/초안 생성 워커

## 목표
- 외부 소스를 주기 수집하고 AI 기사 초안을 생성해 CMS 검토 큐(`desk_review`)로 안전하게 송고한다.

## 범위
- 소스 수집기(RSS/공지/보도자료)
- 중복 감지(`source_hash`)
- LLM 기반 초안/제목 3개/카테고리 생성
- 근거 링크 및 인용 메타 저장
- 실패 재시도/데드레터 큐

## 제외 범위
- 자동 발행
- 포털 송고

## 파이프라인 단계
1. Ingestion: 소스 수집 및 정규화
2. Deduplication: 중복/유사 중복 필터
3. Enrichment: 요약, 엔터티 추출(선택)
4. Generation: 초안/제목/카테고리 생성
5. Validation: 금칙어/길이/형식 검증
6. Push: CMS API로 `desk_review` 송고

## 품질/안전 규칙
- AI 결과는 기본적으로 미검증 상태로 표시
- 모든 초안에 `citations_json` 저장
- 사실 단정 문장 비율/출처 불명 문장 비율 점검(룰 기반)
- 모델 라우팅: 분류/제목 저비용 모델, 본문 고성능 모델

## 작업 항목
1. 소스 어댑터 인터페이스 정의
2. 수집 스케줄러/큐 구성
3. 프롬프트 템플릿 버전 관리
4. LLM 결과 파서/검증기 구현
5. CMS Push API 연동 + 멱등성 처리
6. 실패 재시도/데드레터/알림 구성

## 완료 기준 (DoD)
- 지정 소스에서 일일 목표 건수 이상 자동 초안 생성
- 중복 송고율 목표 이하
- 실패 건 재처리 플로우 동작
- 초안별 근거/모델/프롬프트 버전 추적 가능

## 검토 체크리스트
- 원문이 변경/삭제된 경우 처리 정책이 있는가
- 크롤링 차단(robots/차단 응답) 대응이 있는가
- API 제한 초과 시 백오프가 동작하는가
- 환각 리스크를 감지/표시하는가
